# 修复贴纸翻转功能，只翻转图片不翻转按钮

## 问题分析

当前的翻转实现是通过在整个 `.sticker-wrapper` 容器上应用 `scaleX(var(--f))` 变换，导致：

* 图片被翻转

* 边框被翻转

* 按钮位置和方向也被翻转

## 解决方案

修改 CSS 结构，将翻转效果只应用到图片上，而不是整个容器，同时更新按钮的补偿样式。

## 具体修改

### 1. 修改 style.css

* **修改贴纸容器样式**：从 `.sticker-wrapper` 的 transform 属性中移除 `scaleX(var(--f))`

* **添加图片翻转样式**：为 `.sticker-img` 添加 transform 属性，包含 `scaleX(var(--f))`

* **更新按钮补偿样式**：从 `.handle` 和 `.sticker-control-btn` 的 transform 属性中移除 `scaleX(var(--f))`

### 2. 保持 script.js 不变

* 翻转逻辑不需要修改，因为它仍然需要更新 `--f` CSS 变量

* 按钮的点击事件处理逻辑保持不变

## 预期效果

* 点击翻转按钮时，只有图片会水平翻转

* 边框保持不变，不会被翻转

* 按钮位置和方向保持不变，不会随图片一起翻转

* 其他功能（拖拽、缩放、旋转）保持正常工作

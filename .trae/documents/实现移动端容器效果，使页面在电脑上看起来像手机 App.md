# 实现移动端容器效果

## 1. 添加移动端容器

* 在 `<body>` 标签内添加一个新的容器元素，设置 `max-w-[500px]`、`mx-auto`、`w-full`、`min-h-screen` 等样式

* 给容器添加细边框（`border`）和阴影（`shadow`），使其看起来像手机屏幕

* 确保容器在移动端自动撑满宽度

## 2. 调整引导页布局

* 修改 `#layer-splash` 元素，使其定位在容器内而不是全屏

* 调整背景图的显示方式，确保其只在容器内显示

* 修复「响应呼唤」按钮的位置，使其位于容器底部区域

## 3. 修改其他页面图层

* 调整 `#layer-menu`、`#layer-identity`、`#layer-journal` 等图层的定位和样式

* 确保所有页面都限制在 500px 的容器内

* 保持原有的页面切换逻辑不变

## 4. 响应式优化

* 确保在真正的手机访问时，容器能自动撑满 100% 宽度

* 调整容器的边框和阴影，使其在不同屏幕尺寸下都能良好显示

* 测试各种屏幕尺寸下的显示效果

## 5. 检查 SPA 切换逻辑

* 确保点击「响应呼唤」后，引导层会平滑消失

* 确保后续的手账页/卡面页选择界面能正确显示

* 确保所有页面切换动画和过渡效果正常工作

## 技术实现

* 使用 Tailwind CSS 类来实现容器的样式和响应式设计

* 调整现有的 CSS 样式，确保背景图和图层正确显示

* 保持现有的 JavaScript 逻辑不变，只修改 HTML 结构和 CSS 样式

## 预期效果

* 在电脑浏览器上，页面会显示为一个居中的、带有边框的 500px 宽容器，看起来像手机 App

* 在真正的手机上，容器会自动撑满屏幕宽度

* 所有页面元素都会在容器内正确显示，不会溢出

* 页面切换逻辑保持不变，仍然平滑流畅
